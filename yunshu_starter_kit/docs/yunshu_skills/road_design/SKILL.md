# 目的
协助爸爸完成城市道路、交通工程的规划、设计与绘图任务。确保所有输出符合国家规范（GB/CJJ）及项目特定规则。

# 核心能力
1.  **规范查询**: 快速检索车道宽度、路面结构、交通标志等标准数据。
2.  **图纸审查**: 分析 CAD (DXF/DWG) 文件的图层、块引用和几何关系。
3.  **辅助绘图**: 生成标准化的绘图脚本或直接建议修改参数。

# 引用资源 (Resources)
在执行任务时，**必须** 严格遵循“规范优先”原则，按顺序参考以下文件：
1.  **设计规则库**: [design_rules.md](./resources/design_rules.md) (最高优先级，包含项目强制规则)
2.  **国家规范汇总**: (建议在项目根目录维护一份规范汇总索引)
3.  **图框模块化规范**: [drawing_frame_module.md](./resources/drawing_frame_module.md)

# 常用指令 (Instructions)

## 0. 任务启动前 (Pre-Flight Check)
**必须执行**:
1.  **检索规则**: 根据任务关键词 (如“横坡”、“路灯”) 搜索 `design_rules.md`。
2.  **确认参数**: 提取相关规则 ID，确认设计值。
3.  **规划路径**: 如果规则库缺失，必须先询问爸爸或查阅国家规范，更新规则库后再执行。

## 1. 道路结构设计
当爸爸要求设计路面结构时：
1.  **检查规则**: 确认结构层延伸方式、材料厚度等项目特例。
2.  **检查标准模板**: 如果项目有预设模板，优先使用。
3.  **输出**: 明确列出各层材料、厚度及延伸宽度。

## 2. 交通设施布置
当涉及标志标线时：
1.  **参考 GB 5768**: 确认标志尺寸、颜色和字体。
2.  **检查路缘石**: 确认路缘石尺寸及外露高度。

## 3. 图纸自检流程
在提交设计成果前，执行以下检查：
- [ ] **图层**: 是否使用了正确的图层名称？
- [ ] **比例**: 绘图比例是否正确？
- [ ] **坐标**: 是否使用了统一的坐标系？
- [ ] **连接**: 线条是否闭合？是否有悬空线？

# 错误处理
如果发现图纸不符合规范：
1.  指出具体违反的规范条款。
2.  提供修正建议。
